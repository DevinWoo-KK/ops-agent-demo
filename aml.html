什么是反洗钱（AML）

洗钱分三个阶段：放置（现金进入银行）→ 分层（多账户流转隐匿来源）→ 整合（回流为合法资金）。

银行的 AML 系统靠规则引擎筛查交易——单日现金超5万就报警。结果是：每天几千条告警，99% 是误报，合规人员淹没在假警报中。真正的可疑交易反而容易漏掉。

传统规则的局限

规则引擎能做的

单笔金额超阈值 → 告警
与黑名单账户交易 → 告警
短期高频交易 → 告警

规则引擎做不了的

拆分成多笔小额规避阈值
换不同网点分散存入
多层账户循环流转洗白
识别新型洗钱手法（NFT/混币）

Agent 的价值: 不是替代规则引擎，而是接在规则引擎后面——告警来了，Agent 自动调取关联交易、图谱分析资金链路、核查制裁名单、检索历史案例，然后给出可解释的判断。合规人员只需要复核 Agent 标为"可疑"的案例。

Agent 设计思路

8 个调查工具

transaction_query

查询指定账户的交易流水，含统计摘要（总笔数/总金额/对手方/夜间占比）

graph_analysis

追踪资金流向，发现闭环路径、共享身份、社区网络

rule_engine

逐条检查 FATF/央行 AML 规则，含法规条款引用

risk_scorer

综合评分 0-100，分项评估行为异常/关联风险/黑名单/历史

sanction_check

核查 OFAC/UN 制裁名单，支持模糊匹配防拼写变体

case_retrieval

向量检索相似历史案例，提供参考依据

news_search

检索客户/企业负面舆情

report_generator

生成符合监管格式的 SAR 报告草稿

Human-in-the-Loop

Agent 永远只做分析和建议。账户冻结、SAR 上报等具有法律效力的操作，必须合规官签批。Agent 的完整推理链可展开审计——每一步用了什么工具、什么数据、得出什么结论。

Agent 自动完成

调取交易 · 图谱分析 · 规则检查 · 制裁核查 · 风险评分 · 案例检索 · SAR 草稿生成

合规官决策

确认制裁名单身份 · 决定是否上报央行 · 冻结/限制账户 · 驳回或修正 Agent 结论
